## 💡 비용 변수 5가지 — 냉정한 평가

| 변수 | 영향도 | 현실적 판단 |
|:---|:---:|:---|
| **OCR 모듈 Expo 호환** | 🔴 높음 | react-native-mlkit이 Expo config plugin으로 작동하는지 Day 4에 반드시 검증. 실패 시 → expo-camera 촬영 후 이미지를 직접 ML Kit API로 처리하는 우회 필요. 최악의 경우 +3~5일 추가 공수 |
| **성분 사전 품질** | 🔴 높음 | 200개 성분 × aliases × 한/영 매핑 = 실제 데이터 정리 작업이 3~5일 소요. 개발자에게 맡기면 ₩150만~200만 추가. **직접 하면 ₩0** |
| **공유 카드 퀄리티** | 🟡 중간 | 동적 캔버스(자유 레이아웃) vs 고정 템플릿(3~5종). 고정 템플릿이 개발 공수 50% 절감 + 디자인 일관성 유지. **고정 템플릿 확정 추천** |
| **다국어 카피** | 🟡 중간 | 의료적 뉘앙스 회피가 핵심. "This ingredient is dangerous" ❌ → "This combination may cause irritation" ✅. 영문 카피 전문가 검수 ₩50만~100만. 또는 ChatGPT/Claude로 초안 → 원어민 1회 감수 |
| **QA 범위** | 🟡 중간 | 카메라/OCR은 기기별 편차 큼. 최소 iPhone 2종 + Android 2종 테스트 필수. 크라우드 테스트(Testlio 등) 쓰면 ₩50만~100만, 직접 하면 ₩0 |

## ✅ 절감 가능 vs ❌ 절대 줄이면 안 되는 것

### 절감 가능 (직접 하면 ₩0)

| 항목 | 예상 절감 | 방법 |
|:---|---:|:---|
| 성분 사전 200개 JSON | ₩150만~200만 | CosIng DB + 식약처 데이터 기반, 직접 정리 |
| 다국어 카피 초안 | ₩30만~50만 | AI로 초안 생성 → 원어민 최종 감수만 외주 |
| 공유 카드 디자인 | ₩50만~100만 | 고정 템플릿 3종 → Figma에서 직접 디자인 |
| QA | ₩50만~100만 | 본인 + 주변인 테스트 (최소 4기종) |
| EAS 배포 설정 | ₩30만~50만 | Expo 문서 따라 직접 설정 |
| **합계** | **₩310만 ~ ₩500만** | |

### 절대 줄이면 안 되는 것

| 항목 | 이유 |
|:---|:---|
| **OCR 구현 품질** | "사진 찍으면 성분이 나온다" = 앱의 첫인상. 여기서 실패하면 D1 리텐션 붕괴 |
| **디자인 (최소 수준)** | 공유 카드가 예쁘지 않으면 바이럴 루프 작동 안 함. 최소 ₩100만~200만은 투자 |
| **Apple Developer 계정** | 앱스토어 출시 필수. 절감 불가 |
| **영문 카피 최종 감수** | "의료 조언"으로 오해받으면 앱스토어 리젝 + 법적 리스크 |

## ⚠️ RevenueCat 비용 — 오해 정리

리뷰에서 "결제 연동 시 추가 300만~1,000만원"이라고 했는데, 이건 **자체 결제 시스템 구축** 비용입니다.

**RevenueCat을 쓰면:**

| 항목 | 비용 |
|:---|:---|
| 기본 요금 | **₩0** (월 매출 $2,500 미만 무료) |
| 초과 시 | 매출의 1~2.5% |
| 구현 공수 | 2~3일 (SDK 설치 + Paywall UI + entitlement 체크) |

Apple/Google 수수료(15~30%)는 별도지만, 이건 어떤 결제 방식이든 동일합니다.
결론: **RevenueCat = MVP에서 결제 비용 ₩0, 구현 공수 최소화.**
